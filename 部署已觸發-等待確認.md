## 🚨 如果部署失敗

### 可能的情況

#### 情況 A: Vercel 沒有檢測到變更
**症狀**: Vercel Dashboard 沒有新的部署記錄

**解決**:
```bash
# 再次手動觸發
cd /home/user/webapp
git commit --allow-empty -m "chore: Force deployment"
git push origin main
```

或在 Vercel Dashboard:
```
Deployments → ... → Redeploy
```

---

#### 情況 B: 構建失敗（Error 狀態）
**症狀**: Vercel 顯示紅色錯誤狀態

**解決**:
1. 點擊失敗的部署
2. 查看 "Build Logs"
3. 找到錯誤信息（紅色文字）
4. 截圖發給我，我會幫您解決

**本地測試**:
```bash
cd /home/user/webapp
npm run build
```
如果本地成功，可能是 Vercel 環境問題。

---

#### 情況 C: 部署成功但顯示舊版本
**症狀**: Vercel 顯示 "Ready" 但網站還是舊的

**解決**:
1. 清除瀏覽器緩存（Ctrl+Shift+Delete）
2. 使用無痕模式測試（Ctrl+Shift+N）
3. 硬刷新網站（Ctrl+Shift+R）
4. 清除 Vercel CDN 緩存:
   ```
   Vercel Dashboard → ... → Clear Cache
   ```
